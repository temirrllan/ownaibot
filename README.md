# Telegram userbot с ИИ-фильтром новостей

Userbot работает от имени вашего **личного** аккаунта Telegram (MTProto через
Telethon), читает все каналы, на которые вы подписаны, прогоняет каждое
сообщение через LLM (OpenAI `gpt-4o-mini`) и пересылает подходящие в
**Избранное (Saved Messages)** с краткой подписью «откуда и почему».

Обычный Bot API так не умеет — бот видит только те чаты, куда его добавили.
Поэтому здесь именно userbot.

## Возможности

- **Стартовая сводка (backfill)** — при первом запуске проходит по всем каналам
  за последние `BACKFILL_HOURS` часов (по умолчанию 24), фильтрует и присылает
  подходящее. После завершения ставит маркер `backfill.done` — повторно сводка
  не запускается.
- **Реальное время** — каждое новое сообщение в любом канале сразу проверяется
  и при необходимости пересылается.
- **Дедупликация** — одинаковые/похожие новости (по нормализованному отпечатку:
  без ссылок, пунктуации, в нижнем регистре) не присылаются дважды, даже из
  разных каналов. Хэши хранятся в `dedup.txt`, размер файла ограничен (`DEDUP_MAX`).
- **Экономия запросов** — слишком короткие сообщения (< `MIN_LEN`) в ИИ не идут.
- Реагирует **только на каналы-broadcast**, личные чаты и группы игнорирует.
- Бережёт лимиты Telegram (паузы в backfill, обработка `FloodWait`).
- Переживает обрывы связи: `run_until_disconnected` + внешний цикл перезапуска
  + `Restart=always` в systemd.

## ⚠️ Безопасность — прочитайте обязательно

- **Файл сессии (`userbot.session`) = полный доступ к вашему аккаунту.**
  Кто получит этот файл — получит ваш Telegram. Никогда не выкладывайте его в
  git, облако, чаты; не передавайте никому. На сервере держите права `chmod 600`.
- **Ключи не хардкодим.** API_ID / API_HASH / OPENAI_KEY храним в файле `.env`
  (см. `.env.example`) или в переменных окружения. `.env` и `*.session` уже в
  `.gitignore`.

## ⚠️ Риски userbot (серая зона правил Telegram) и как их снизить

Автоматизация действий личного аккаунта формально не приветствуется правилами
Telegram. Теоретический риск — ограничение или блокировка аккаунта. Как снизить:

- **Только чтение и пересылка в своё Избранное.** Не рассылаем сообщения другим,
  не вступаем/выходим из каналов массово, не парсим участников — именно такие
  действия чаще всего триггерят анти-спам.
- **Низкая интенсивность.** В backfill стоят паузы (`BACKFILL_DELAY_SEC`),
  обрабатывается `FloodWaitError` (ждём столько, сколько просит Telegram).
- **Не на основном/критичном аккаунте**, если переживаете за него: можно завести
  отдельный номер.
- **Никакого «накручивания» и спама** — этот бот только фильтрует вашу же ленту.

Риск при таком сценарии (пассивное чтение + self-forward) низкий, но не нулевой.

## Установка и первый запуск (локально)

Сначала запускаем **локально**, чтобы один раз ввести код подтверждения из
Telegram и создать файл сессии. Потом сессию переносим на сервер.

### 1. Получить api_id / api_hash

1. Откройте <https://my.telegram.org> и войдите по своему номеру.
2. Раздел **API development tools** → создайте приложение (любое название).
3. Скопируйте **api_id** (число) и **api_hash** (строка).

### 2. Установить зависимости

```bash
git clone <ваш-репозиторий> ownaibot   # или просто скопируйте файлы
cd ownaibot

python3 -m venv .venv
source .venv/bin/activate               # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настроить .env

```bash
cp .env.example .env
nano .env        # вставьте API_ID, API_HASH, OPENAI_KEY
```

Интересы (`MY_INTERESTS`) удобнее править прямо в `userbot.py`.

### 4. Первый запуск — вход в Telegram

```bash
python userbot.py
```

Telethon спросит:

- номер телефона (в формате `+7...`),
- код из официального приложения Telegram,
- пароль two-step verification, если он включён.

После входа создастся файл `userbot.session`. Сразу пойдёт стартовая сводка,
затем бот перейдёт в режим реального времени. Остановить — `Ctrl+C`.

## Перенос на VPS (Ubuntu/Debian) и автозапуск 24/7

### 1. Подготовить сервер

```bash
sudo apt update && sudo apt install -y python3 python3-venv git
mkdir -p ~/ownaibot && cd ~/ownaibot
# скопируйте сюда: userbot.py, requirements.txt, .env, telegram-userbot.service
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

### 2. Перенести файл сессии

Скопируйте созданный локально `userbot.session` на сервер (например, по scp):

```bash
scp userbot.session ubuntu@SERVER_IP:~/ownaibot/
ssh ubuntu@SERVER_IP 'chmod 600 ~/ownaibot/userbot.session ~/ownaibot/.env'
```

Так на сервере не придётся вводить код повторно. **Файл сессии — секрет!**

### 3. Установить systemd-сервис

Откройте `telegram-userbot.service` и поправьте `User`, `WorkingDirectory`,
`ExecStart`, `EnvironmentFile` под свои пути (пример рассчитан на пользователя
`ubuntu` и каталог `/home/ubuntu/ownaibot`).

```bash
sudo cp telegram-userbot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now telegram-userbot
```

### 4. Логи и управление

```bash
# живой просмотр логов
journalctl -u telegram-userbot -f

# последние 100 строк
journalctl -u telegram-userbot -n 100 --no-pager

# статус / перезапуск / стоп
systemctl status telegram-userbot
sudo systemctl restart telegram-userbot
sudo systemctl stop telegram-userbot
```

## Настройки (в начале `userbot.py` / в `.env`)

| Параметр | Назначение | По умолчанию |
|---|---|---|
| `API_ID`, `API_HASH` | авторизация Telegram | — (обязательно) |
| `OPENAI_KEY` | ключ OpenAI | — (обязательно) |
| `OPENAI_MODEL` | модель фильтра | `gpt-4o-mini` |
| `MY_INTERESTS` | описание интересов своими словами | см. код |
| `TARGET` | куда слать (`"me"` = Избранное) | `me` |
| `BACKFILL_HOURS` | глубина стартовой сводки, ч | `24` |
| `BACKFILL_LIMIT_PER_CHANNEL` | макс. сообщений на канал в сводке | `50` |
| `MIN_LEN` | мин. длина сообщения для ИИ | `60` |
| `DEDUP_MAX` | макс. хранимых отпечатков | `5000` |
| `BACKFILL_DELAY_SEC` | пауза между сообщениями в сводке | `0.7` |

## Файлы состояния (создаются автоматически в рабочей папке)

- `userbot.session` — сессия Telegram (**секрет!**).
- `dedup.txt` — отпечатки уже пересланных новостей.
- `backfill.done` — маркер выполненной стартовой сводки.

Чтобы **повторить стартовую сводку** — удалите `backfill.done`.
Чтобы **сбросить дедупликацию** — удалите `dedup.txt`.
